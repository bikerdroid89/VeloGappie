# Contributing to VeloGappie

Thanks for your interest in contributing!

## Reporting bugs

Open an [issue](../../issues/new?template=bug_report.yml) with:
- Your bike model and variant
- Android version and phone model
- Steps to reproduce the problem

## Feature requests

Open an [issue](../../issues/new?template=feature_request.yml) describing the feature and why it would be useful.

## Bike compatibility

If you have a Veloretti V2 bike, we'd love to hear whether VeloGappie works with it. Open a [compatibility report](../../issues/new?template=bike_compatibility.yml) with your bike model and what works / what doesn't.

## Code contributions

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes and commit
4. Open a pull request with a clear description

Please open an issue first for larger changes so we can discuss the approach.

## Code style

- Kotlin with Jetpack Compose
- Follow existing patterns in the codebase
- Keep commits focused and descriptive
